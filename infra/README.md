# Infra

## Цель инфраструктуры

Инфраструктура должна быть простой, дешёвой и достаточной для первых клиентов.

## Базовый стек

- `Hetzner`
- `Docker Compose`
- `Caddy`
- `n8n`
- лендинг

## Текущая production-реальность

На живом сервере уже работают:
- `caddy`
- `n8n`
- `scanner`
- `redis`
- `openclaw`

## Правило

Не раздувать infra раньше времени. Каждый новый сервис должен отвечать на один из вопросов:
- помогает ли он получить лида
- помогает ли он обработать лида
- помогает ли он отдать результат клиенту
- помогает ли он не сломать уже работающий поток

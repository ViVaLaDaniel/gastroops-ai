# CRM в Google Sheets

## Зачем это нужно

На старте `Google Sheets` — это не “идеальная CRM”, а быстрый и прозрачный источник правды по лидам.

Для `V1` этого достаточно, если:
- все новые заявки попадают в один лист
- статусы меняются дисциплинированно
- `n8n` пишет только в согласованные колонки

## Минимальная структура

На первом этапе достаточно 4 листов:

1. `Leads`
2. `Clients`
3. `Orders`
4. `Tasks`

Для первого workflow сейчас обязателен только лист `Leads`.

## Лист Leads

Колонки должны быть такими:

| Колонка | Назначение |
| --- | --- |
| `lead_id` | Уникальный ID лида |
| `created_at` | Дата и время создания |
| `source` | Источник лида: `website`, `email`, `manual`, `outreach` |
| `name` | Имя контакта |
| `restaurant_name` | Название ресторана или проекта |
| `city` | Город |
| `language` | Язык общения |
| `email` | Email |
| `phone` | Телефон или WhatsApp |
| `website` | Сайт |
| `instagram` | Instagram, если есть |
| `menu_url` | Ссылка на текущее меню |
| `requested_service` | Что запросил клиент |
| `pain` | Боль или задача клиента |
| `budget_hint` | Если бюджет известен |
| `status` | Статус лида |
| `next_action` | Следующий шаг |
| `owner` | Кто ведёт лида |
| `last_contact_at` | Последний контакт |
| `notes` | Свободные заметки |

## Статусы

Рабочие статусы для `V1`:

- `new`
- `contacted`
- `interested`
- `audit_sent`
- `call_booked`
- `proposal_sent`
- `paid`
- `in_progress`
- `delivered`
- `lost`

## Правила

- Один лид = одна строка.
- `Leads` — стартовая точка правды для всех новых заявок.
- Не создавать отдельные таблицы “на всякий случай”.
- Если лид стал клиентом, его не удалять из `Leads`, а переводить дальше по статусу.

## Что должно заполняться автоматически через n8n

Через workflow должны автоматически заполняться:

- `lead_id`
- `created_at`
- `source`
- `name`
- `restaurant_name`
- `city`
- `language`
- `email`
- `phone`
- `website`
- `menu_url`
- `requested_service`
- `pain`
- `status`
- `next_action`
- `owner`

## Что можно дополнять вручную

Вручную потом можно вносить:

- `instagram`
- `budget_hint`
- `last_contact_at`
- `notes`

## Практическое правило

Новички теряют деньги не потому, что у них слабый `AI`, а потому что они теряют лидов в хаосе.

Поэтому эта таблица нужна не “для красоты”, а чтобы каждый лид имел:
- статус
- владельца
- следующий шаг
- историю движения

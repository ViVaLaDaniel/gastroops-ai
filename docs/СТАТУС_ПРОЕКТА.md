# Статус Проекта

## Состояние на сейчас

Проект находится в стадии `операционная подготовка + сборка V1`.

Базовая production-инфраструктура уже живая, но сам продуктовый слой ещё не доведён до первого полноценного клиентского workflow.

## Что уже работает

- домен и сабдомены указывают на `Hetzner`
- `n8n` доступен по `https://n8n.n8ndo.es`
- `OpenClaw` доступен по `https://openclaw.n8ndo.es`
- `scanner` доступен по `https://scanner.n8ndo.es`
- `Caddy` маршрутизирует домены и выдаёт `HTTPS`
- репозиторий проекта опубликован в `GitHub`: `ViVaLaDaniel/gastroops-ai`
- подготовлена схема первого intake workflow
- подготовлена CRM-структура для `Google Sheets`
- лендинг уже содержит форму под lead intake
- глобальная `MCP`-конфигурация переведена на hosted `Notion` и hosted `Linear`

## Что ещё не закончено

- repo-local инфраструктура не полностью совпадает с production
- `Google Sheets` и `Telegram` credentials ещё не привязаны в `n8n`
- workflow лидов ещё не собран и не протестирован внутри `n8n`
- шаблоны клиентской работы ещё не оформлены как полноценный пакет
- внешний content-sync в `Notion` и `Linear` ещё не был выполнен в этом runtime

## Главный риск

Потратить время на технику и документацию, но не запустить рабочий поток получения и обработки лида.

## Главная цель ближайшего этапа

Дособрать один полный путь:

`лид -> CRM -> уведомление -> ответ -> brief -> delivery`

# Личная Операционная Система

## Зачем тебе это нужно

Если ты хочешь строить проекты, деньги и жизнь серьёзно, тебе нужна не “ещё одна заметка”, а система.

Без системы будет так:
- идеи отдельно
- задачи отдельно
- документы отдельно
- история теряется
- проекты не доводятся

Система нужна, чтобы у тебя была вторая память.

## Главный принцип

Каждая система должна отвечать только за свою роль.

Нельзя делать так, чтобы:
- `Notion` был и CRM, и архивом, и технической вики, и дневником, и board, и всем сразу
- `Obsidian` был просто свалкой markdown
- `Linear` использовался как личный дневник

## Правильное разделение

### `Notion`

Это твоя главная human-facing база жизни и бизнеса.

Там должны жить:
- проекты
- задачи
- области жизни
- люди / контакты
- сессии
- ссылки на документы
- high-level planning

### `Obsidian`

Это твоя локальная knowledge base.

Там должны жить:
- полная история
- архитектура
- SOP
- handoff notes
- research
- длинные документы
- всё, что ты хочешь контролировать локально

### `Linear`

Это execution system.

Там должны жить:
- только активные задачи
- blockers
- прогресс
- статусы

### `Supabase / Agent-Workspace-DB`

Это machine memory для агентов и automation.

Там должны жить:
- closeout
- next actions
- structured sessions
- technical memories

## Как должна выглядеть твоя жизнь в системе

### Уровень 1. Life Areas

Главные области:
- здоровье
- деньги
- работа
- проекты
- обучение
- отношения
- быт

### Уровень 2. Projects

Каждый серьёзный проект должен иметь:
- одну main page в `Notion`
- один project space в `Obsidian`
- одну active execution линию в `Linear`
- durable memory в `Supabase`

### Уровень 3. Tasks

Каждая задача должна отвечать на 3 вопроса:
- что нужно сделать
- зачем это нужно
- какой следующий конкретный шаг

### Уровень 4. Sessions

Каждая рабочая сессия должна фиксировать:
- что сделал
- что не доделал
- blocker
- следующий шаг
- если нужно: настроение / состояние / энергия

## Что должно быть в Notion

Минимальные базы:

1. `Projects`
2. `Tasks`
3. `Sessions`
4. `Areas of Life`
5. `Knowledge / Documents`
6. `People / Contacts`
7. `Ideas`

## Что должно быть в Obsidian

Минимальная структура:

```text
Task/Codex Global Hub/
  00 Global Task Hub.md
  Projects/
    Project Name/
      Architecture.md
      Tasks.md
      System.md
      History/
      SOP/
```

## Что должно быть в каждом проекте

Для каждого проекта обязательно:
- `Status.md`
- `Architecture.md`
- `Tasks.md`
- `System.md`

Желательно:
- `SOP/`
- `History/`
- `Sessions/`

## Где теперь лежат шаблоны

В `Obsidian` уже созданы:

- `Task/Codex Global Hub/System/NOTION_СТРУКТУРА.md`
- `Task/Codex Global Hub/System/ПРАВИЛА_СИНХРОНИЗАЦИИ.md`
- `Task/Codex Global Hub/System/ШАБЛОН_РАБОЧЕЙ_СЕССИИ.md`
- `Task/Codex Global Hub/Projects/_Project Template/`

## Как не утонуть в документации

Не всё документировать одинаково глубоко.

Правило:
- если это влияет на деньги, качество или повторяемость — документировать
- если это одноразовая мелочь — не раздувать

## Как думать как взрослый бизнесмен

Система не должна быть красивой ради красоты.

Она должна:
- уменьшать хаос
- ускорять принятие решений
- сохранять историю
- делать проекты повторяемыми
- помогать зарабатывать

## Твой рабочий ритм

На практике это должно выглядеть так:

1. Утром смотришь `Notion` и active tasks
2. Работаешь по текущему проекту
3. Длинный контекст и решения пишешь в `Obsidian`
4. Активное выполнение ведёшь через `Linear`
5. В конце сессии делаешь closeout

## Самое важное правило

Если система не помогает тебе:
- быстрее видеть реальность
- быстрее принимать решения
- быстрее двигать проекты
- лучше помнить историю

значит это не система, а декорация.

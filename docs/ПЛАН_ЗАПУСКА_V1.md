# План Запуска V1

## Цель

Не построить идеальную платформу, а запустить рабочую систему для первых клиентов.

## Этап 1. Упаковать оффер

- зафиксировать 3 стартовые услуги
- зафиксировать цены
- зафиксировать основной CTA
- подготовить 2–3 демо-материала

## Этап 2. Довести инфраструктуру

- проверить production-стек
- описать deploy flow
- синхронизировать repo-local stack с production-реальностью

## Этап 3. Настроить intake

- форма сайта
- запись в `Google Sheets`
- уведомление в `Telegram`
- базовый статус лида

## Этап 4. Настроить communication flow

- `Gmail -> draft -> approval`
- шаблон первого ответа
- шаблон brief

## Этап 5. Настроить delivery flow

- папка клиента
- шаблон mini audit
- шаблон proposal
- шаблон menu pack
- шаблон tech cards / costing

## Этап 6. Запустить продажи

- собрать целевой список ресторанов
- начать outreach
- проводить mini audit
- довести первые диалоги до оплаты

## Критерий готовности V1

V1 считается готовым, когда есть полный рабочий путь:

`лид -> CRM -> Telegram -> ответ -> brief -> delivery -> оплата`
